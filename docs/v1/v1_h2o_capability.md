## 1. 结论声明

**V1 对 H2O zero/span 业务闭环的正式能力结论：NOT_SUPPORTED**

### 1.1 "结构存在"与"业务闭环不支持"的区别

| 概念 | 含义 | V1状态 |
|------|------|--------|
| H2O路由/点位结构存在 | 代码中存在H2O路由分支、H2O点位解析、H2O相关配置项 | **是** |
| H2O zero/span业务闭环 | 从点表→路由→稳态→采样→拟合→写回的完整业务闭环，与CO2对等 | **否** |

**关键区别**：V1 代码中存在 H2O 相关的路由分支和点位结构，但这仅表示"代码结构上支持 H2O 点位的识别和部分执行"，不等于"存在与 CO2 对等的完整业务闭环"。

## 2. 点表解析环节对比

**代码定位**: `data/points.py:341`

| 对比项 | CO2 | H2O |
|--------|-----|-----|
| 列解析 | 直接数值读取，支持向下继承 | 文本正则解析（`XX℃（湿度发生器）`、`XX%（湿度发生器）`、`XX℃（露点温度）`、`XXmmol/mol`） |
| 向下继承 | CO2列空单元格沿用上一条有效值 | H2O carry-forward 可选开启（`h2o_carry_forward`配置） |
| off标记 | 无 | 支持 `——`/`—`/`N/A`/`NA`/`NONE`/`无` 等标记显式关闭H2O上下文 |
| 零下温度 | 正常解析 | 强制清除H2O相关字段（零下仅走气路） |
| 互斥关系 | 有co2_ppm时清除H2O字段 | 有hgen_temp+hgen_rh时清除co2_ppm和co2_group |
| 完整性 | **完整** | **部分支持**（文本解析+carry-forward+off标记+零下强制清除） |

## 3. 路由执行环节对比

**代码定位**: `workflow/runner.py`（`_run_point`、`_run_co2_point`、`_run_h2o_point`）

| 对比项 | CO2 | H2O |
|--------|-----|-----|
| 路由分支 | `_run_co2_point()` | `_run_h2o_point()` |
| 完整流程 | 温箱等待→开路浸泡→封路→控压→封压后露点门禁→采样→系数写回 | 温箱等待→湿度发生器稳定→开路露点对齐→封路→采样 |
| 压力控制 | PACE5000完整控压闭环 | 无独立压力控制 |
| 封压后门禁 | 完整（span+slope+反弹+物理QC） | 参数宽松，无独立验证证据 |
| 系数写回 | 完整闭环（auto_fit→SENCO→GETCO→回滚） | **无独立写回路径** |
| 完整性 | **完整闭环** | **结构存在但无完整闭环** |

### 3.1 CO2完整流程

```
温箱等待 → 开路浸泡 → 封路 → 控压 → 封压后露点门禁 → 采样 → 系数写回
```

### 3.2 H2O部分流程

```
温箱等待 → 湿度发生器稳定 → 开路露点对齐 → 封路 → 采样
                                                    ↑
                                            无系数写回
                                            无独立压力控制
                                            无完整门禁验证
```

## 7. 配置边界

### 7.1 H2O能力配置

| 配置路径 | 值 | 行号 | 说明 |
|---------|----|------|------|
| coefficients.h2o_zero_span.status | "not_supported" | config.py:320 | H2O能力状态声明 |

### 7.2 生效逻辑

`require_v1_h2o_zero_span_supported()`（config.py）：

- 读取 `coefficients.h2o_zero_span.status`
- 若值不为 `"supported"`，抛出 `RuntimeError`
- 当前默认值为 `"not_supported"`，因此调用此函数会始终抛出异常

### 7.3 H2O摘要选择策略

`h2o_summary_selection.py` 提供了 H2O 摘要选择策略的配置，但这是用于离线后处理的选择逻辑，不是运行时业务闭环的一部分。

## 8. V2补齐参考

### 8.1 V1中H2O缺失的具体环节

| 环节 | 缺失内容 | V2需!补齐 |
|------|---------|----------|
| 路由执行 | H2O无独立压力控制 | H2O压力控制闭环 |
| 路由执行 | H2O无系数写回 | H2O系数写回闭环 |
| 稳态判定 | H2O封压后门禁无独立验证 | H2O门禁验证测试 |
| 稳态判定 | H2O无反弹守护 | H2O反弹守护 |
| 稳态判定 | H2O无物理QC | H2O物理一致性QC |
| 系数写回 | H2O无拟合路径 | H2O拟合+SENCO+GETCO闭环 |
| 系数写回 | H2O无审计记录 | H2O系数写回审计 |

### 8.2 V2需要补齐的能力清单

1. H2O zero/span 完整业务闭环（点表→路由→稳态→采样→拟合→写回）
2. H2O 独立压力控制（如需要）
3. H2O 封压后露点门禁的独立验证证据
4. H2O 反弹守护和物理QC
5. H2O 系数写回闭环（拟合→SENCO→GETCO→回滚→审计）
6. H2O 能力状态从 `not_supported` 变更为 `supported` 的条件与验证
